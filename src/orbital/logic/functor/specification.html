Problemspezifikation

Ein System soll erschaffen werden, welches die wichtigsten Eigenschaften der folgenden Teilgebiete der Informatik vereint:

  1. logische Programmierung (wie in Prolog)

  2. funktionale Programmierung (wie in Mathematica)

  3. objektorientierte Programmierung (wie in Java)

Am besten sollte das System auf Java basieren, damit wichtige vorhandene Konzepte wie Verteilung und Komponentenarchitektur erhalten bleiben können, sowie bestehende Problemlösungen weiterverwendet werden können. Bei genügender Integration in die Programmiersprache Java kann der Aspekt der objektorientierten Programmierung direkt von Java übernommen werden.

Analyse der notwendigen Schritte

  • Syntax einer Sprache für die zu bearbeitenden Terme entwerfen. Diese sollte sich an die übliche Prädikatendarstellung anlehnen. Festlegen, wie Variablen für die Signatur gekennzeichnet werden sollen. Vorstellbar wäre die Lösung wie in Mathematica mit nachgestelltem _ Zeichen.

  • Geeignete Datenstruktur zur Repräsentation von Termen festlegen.

  • Termparser erstellen, der einen Term in Form eines Strings in seine Komponenten zerlegt und die zugehörige Termrepräsentation erzeugt.

  • Substitutionen ermöglichen, die Teilterme durch andere Teilterme ersetzen, und sowohl an einer Stelle als auch auf den ganzen Term angewandt werden können.

  • Wiederverwendbaren Algorithmus für Robinson-Unifikation erstellen, der einen Unifikator als Substitution liefert. Einschränkung des Algorithmus auf einseitige Unifikatoren (Matcher) sollte möglich sein.

  • Eine geeignete Datenstruktur zur Haltung von Termersetzungsregeln definieren. Genannt „Termersetzungsdatenbasis“.

  • Parser für Termersetzungsregeln schreiben, der eine Datei mit einem Satz Termersetzungsregeln in eine Termersetzungsdatenbasis übersetzt.

  • Mittels Unifikation ein Term-Rewrite-System (TRS) erschaffen, welches in priorisierter Reihenfolge die Termersetzungsregeln aus der Termersetzungsdatenbasis auf einen Eingabeterm anwendet. Diese Substitutionen sollten erschöpfend durchgeführt werden können.

  • Funktionale Semantik als Termersetzungssystem formulieren.

  • Komponente erstellen, die logische Inferenzen auf Prädikaten und Funktionen unter Verwendung des Unifikationsmechanismus durchführen kann. Falls möglich als Termersetzungssystem formulieren.

  • Möglichkeit schaffen, weitere Logiken zu implementieren, die auch von einem Formelparser gelesen werden können sollen.

  • Vereinheitlichung des Zahlenkonzepts: Skalar, Vektor, Matrix, Liste, Symbol, etc.
