Vector graphics file format *.vec:

header:
    j_int     reserved
    j_byte    id    0x3D
    j_short   version f.ex. 0x0109
    j_short   size of header (incl. field "size of header"), without "object count"
    j_short   size of object
    j_short   size of polygon
    j_byte    use polygon sorting?
    j_byte    display with zbuffer usage?
    j_int     offset to first object (rel. to beginning of file)
    .
    .
    .

objectoffset:
    - objects will appear after the header -
    j_short    object count

    - object I in 3D world -
    j_short   polygon count
    j_short   number of vertexes in object
    j_short   x punkt 1
    j_short   y punkt 1
    j_short   z punkt 1
    j_short   x punkt 2
    j_short   y punkt 2
    j_short   z punkt 2
    .
    .
    .

    - polygon J in object -
    j_short   vertex count in polygon 1
    Color     color of polygon 1 (j_int 0xFFrrggbb)
    j_short   vertex #1 in polygon 1
    j_short   vertex #2 in polygon 1
    .
    .
    .



Animation format *.Ani

    
    j_int     reserved
    j_byte    id    0x3A
    j_short   version f.ex. 0x0109
    j_short   size of header (incl. field "size of header")
    j_short   size of command chunk
    j_int     offset to first command (rel. to beginning of file)


commandoffset:
    j_byte    command#:
              1 - move / position
              2 - scale
              3 - rotate
              4 - execute transformation now
              5 - set delaytime
              F - end of initial movement      // all commands up to F will be skipped in loops
    3*j_int   command-specific data:
//  vvvvvv :  fixed-point numbers, 22bit.10bit
    j_fixed  x data     |    j_int int: time in ms  /  number of steps
    j_fixed  y data     |    j_int      0
    j_fixed  z data     |    j_int      0
