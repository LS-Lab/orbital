Configuring Orbital as a Security Provider

For the security provider factories to work in a generic way, you must add the provider to your list of approved security providers. If you prefer to use these generic factories this can be achieved in the following ways:
  • by using the Orbital library setup wizard for configuration. To start the setup wizard, call
      •  java -jar orbital-ext.jar
    If you got Orbital library as a download extension with Java Plug-in 1.3, this wizard should already have been started.
  • statically by editing the java.security file in the lib/security directory of the JVM. There you should add
      •  security.provider.n=orbital.moon.io.cryptix.provider.Orbital
    where n specifies its preference order. The preference order is the order in which providers are searched for requested algorithms (when no specific provider is requested). The lower the number, the higher the preference. 1 is the provider preferred most.
  • Adding a provider can also be done dynamically but transient as needed by calling
      • Security.addProvider(
        new orbital.moon.io.cryptix.provider.Orbital()
      );
    However, this variant requires the
      permission java.security.SecurityPermission "Security.insertProvider.Orbital";
    granted in the local Java policy file for any program that adds a provider dynamically.
